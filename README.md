# ✈️ FlyForecast

A time series forecasting service that predicts monthly international airline passenger volumes using a seasonal ARIMA (SARIMAX) model, served through a lightweight Streamlit interface.

## Overview

FlyForecast forecasts future monthly airline passenger counts on the classic Box-Jenkins Airline Passengers dataset (1949-1960). It walks the full time series modeling lifecycle - stationarity diagnosis, transformation testing, ACF/PACF-driven order selection, baseline vs. seasonal model comparison - and ships the winning model behind a simple web UI for interactive forecasting.

The goal was not just to fit a model, but to justify why a seasonal model was necessary over a non-seasonal one, using measurable error reduction rather than assumption.

## Dataset

- **Source:** `seaborn.load_dataset('flights')`, the well-known Box-Jenkins international airline passengers series
- **Range:** January 1949 to December 1960 (144 monthly observations)
- **Fields:** `year`, `month`, `passengers` (monthly totals, in thousands)
- **Characteristics:** strong upward trend, consistent yearly seasonality, and multiplicative variance (seasonal swings grow as the trend rises)

## Methodology

### 1. Stationarity Diagnosis

The raw series was tested with the Augmented Dickey-Fuller (ADF) test and found to be strongly non-stationary (test statistic approximately 0.82, p-value approximately 0.99, nowhere close to any critical value). Several transformations were then evaluated to identify the best stabilizing approach:

| Transform | ADF Statistic | p-value | Verdict |
|---|---|---|---|
| Raw series | 0.815 | 0.992 | Non-stationary |
| First difference | -2.829 | 0.054 | Borderline, closest single transform |
| Log | -1.717 | 0.422 | Non-stationary |
| Square root | -0.346 | 0.919 | Non-stationary |
| Cube root | -0.810 | 0.816 | Non-stationary |
| Log + sqrt + diff | -2.611 | 0.091 | Borderline |

None of the simple transforms fully stabilized the series on their own, a strong signal that the seasonality itself needed to be modeled explicitly rather than transformed away.

### 2. Order Selection

ACF/PACF plots were generated on both the first-differenced series and the 12-lag (seasonal) differenced series to identify candidate AR/MA and seasonal AR/MA orders before fitting.

### 3. Model Comparison

Data was split chronologically (70/30, no shuffling) to preserve temporal order, and two models were trained and evaluated on held-out RMSE:

| Model | Order | Test RMSE | Notes |
|---|---|---|---|
| ARIMA | (1,1,3) | **42.99** | Captures trend; misses yearly seasonal swings |
| **SARIMAX** | (1,1,1)(1,1,0,12) | **26.85** | Explicitly models 12-month seasonality |

The seasonal model cut test RMSE by approximately 37.5% over the non-seasonal baseline, confirming that seasonality, not just trend, was the dominant structure in the data. SARIMAX(1,1,1)(1,1,0,12) was selected as the production model.

### 4. Forecasting

The final SARIMAX model was refit and used to generate a 5-year forward forecast (1961-1965) to validate that it extrapolates the trend and seasonal cycle sensibly beyond the observed range.

## Model Artifact

The trained model is serialized as `model.pkl`, containing:

```python
{
    "model_fit":  <fitted SARIMAX results object>,
    "first_date": <first timestamp in training data>,
    "last_date":  <last timestamp the model was trained through>,
}
```

`last_date` anchors the app's forecast horizon. The UI computes the number of steps to forecast as the offset between the requested month/year and this date.

## Application

`app.py` is the FlyForecast Streamlit app, with a custom-styled, card-based UI:

- User selects a target month and year (bounded to a 145-year window past the model's last training date)
- The app computes the forecast horizon in months and calls `get_forecast(steps=...)` on the loaded SARIMAX model
- The predicted mean is displayed in a styled result card

> **Note:** the app currently computes a confidence interval (`forecast.conf_int()`) but does not render it. Surfacing the interval alongside the point estimate is a natural next step (see Future Improvements below).

## Getting Started

### Prerequisites

- Python 3.10+
- pip

### Installation

```bash
git clone <your-repo-url>
cd flyforecast
pip install -r requirements.txt
```

> **Heads-up:** `requirements.txt` currently lists only the app's direct runtime dependencies. To re-run the notebook end-to-end you'll also need `statsmodels` and `scikit-learn`, which power the ADF test, ARIMA/SARIMAX models, and RMSE evaluation. Worth adding both to `requirements.txt` for full reproducibility:
> ```bash
> pip install statsmodels scikit-learn
> ```

### Usage

1. Run `Air_Passengers.ipynb` end-to-end to train the model and generate `model.pkl` in the project root.
2. Launch the app:
   ```bash
   streamlit run app.py
   ```
3. Open the local URL Streamlit prints, pick a month and year, and click Predict.
